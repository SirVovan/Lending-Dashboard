# ШАГ 1: АЛГОРИТМ АНАЛИЗА HTML-ФАЙЛА

## 1.1. CSS-переменные

Найди блок `:root { ... }` внутри `<style>`.
Извлеки все `--переменные` и их значения — они понадобятся для стилизации `admin.html`.

Fallback (если `:root` отсутствует):
```
--bg: #1a1a2e; --bg2: #16213e; --bg3: #0f3460;
--accent: #6c63ff; --accent-lt: #a29bfe;
--text: #ffffff; --muted: #aaaaaa;
--border: rgba(255,255,255,0.15); --glass: rgba(255,255,255,0.05);
```

## 1.2. Шрифты

Найди все `<link href="*fonts.googleapis.com*">` и `<link href="*fonts.bunny.net*">` — скопировать в `admin.html` без изменений.

Определи:
- **Основной шрифт** — значение `font-family` у `body` в CSS
- **Заголовочный шрифт** — значение `font-family` у `h1, h2` в CSS

## 1.3. Определение секций

Ищи элементы строго в порядке приоритетов:

1. `<section id="X">` — использовать `id` как ключ секции
2. `<div id="X">` содержащий дочерние `h1` или `h2`
3. Блоки с классами: `.hero`, `.about`, `.features`, `.services`, `.pricing`, `.testimonials`, `.contact`, `.footer`
4. Разделить по `<h2>` — каждый `h2` начинает новую секцию, имя = slug от текста
5. **Fallback:** одна секция `"main"` со всеми текстами

**ПРОПУСТИТЬ:**
- `cookieBanner`, `privacyOverlay`, попапы с `display:none`, модальные окна
- `<nav>` — обработать отдельно как секцию `"nav"`
- `<footer>` — обработать отдельно как секцию `"footer"`

## 1.4. Классификация элементов внутри каждой секции

**СКАЛЯРНЫЙ ТЕКСТ** → атрибут `data-bind="секция.ключ"`:
- `h1`, `h2`, `h3`, `h4` — если нет дочерних блочных элементов (допустимы только `<em>`, `<strong>`, `<span>` с текстом)
- `<p>` — если нет вложенных `div` или `ul`
- Элементы с классами: `.eyebrow`, `.label`, `.tag`, `.badge`, `.caption`, `.subtitle`, `.desc`, `.tagline`
- `<button>` и `<a>` — тексты кнопок CTA (не ссылки-якори `href="#section"`)
- `<span>` — если является единственным текстовым потомком своего родителя

**HTML-ТЕКСТ** → атрибут `data-bind-html="секция.ключ"`:
- `h1`/`h2`/`h3` содержащие `<em>` в середине текста (паттерн: `Текст <em>акцент</em>`)
- `<p>` или `<div>` где `innerHTML` содержит `<strong>`, `<em>`, `<br>`, `<a>` вперемешку с текстом
- Значение в JSON — это `innerHTML` элемента (включая теги)

**ИЗОБРАЖЕНИЕ** → атрибут `data-bind-src="секция.ключ"`:
- Все `<img src="...">` теги
- Ключ = имя файла без расширения + `_url` (пример: `src="img/hero-photo.jpg"` → ключ `hero_photo_url`)
- Значение в JSON = исходный `src` атрибут

**МАССИВ** → атрибут `data-builder="секция.items"` на контейнере:
- Признак: 3 и более элементов с одинаковым CSS-классом подряд внутри одного контейнера
- Примеры классов: `.svc-card`, `.review-card`, `.faq-item`, `.event-card`, `.stat`, `.step`, `.team-member`
- FAQ-паттерн: элементы с `onclick="toggleFaq(this)"` или `data-toggle`
- Добавить атрибут на **контейнер** этих элементов, не на сами элементы

**ИГНОРИРОВАТЬ:**
- Всё содержимое внутри `<svg>` тегов
- Inline стили (`style="..."`)
- Содержимое `<script>` тегов
- Тексты короче 2 символов
- Технические `data-*` атрибуты
- Числа-счётчики в canvas-анимациях
- Иконки-символы (стрелки, буллеты типа `•`, `→`, `▸`)

## 1.5. Правила именования ключей

- Только латиница, `snake_case`
- Если в секции несколько заголовков: `title` → `subtitle` → `title_3`
- Стандартные имена:
  - `title` — основной заголовок секции
  - `eyebrow` / `subtitle` — надпись над заголовком или подзаголовок
  - `text` / `description` / `desc` — тексты абзацев
  - `cta_text` / `btn_label` — тексты кнопок
  - `pill_text` — текст в пилюле/бейдже
- Внутри массива: ключ = последнее слово CSS-класса без префикса
  (`.svc-name` → `name`, `.review-text` → `text`, `.faq-q` → `q`)
- Изображения: `имя_файла_url` (`.` и `-` заменить на `_`)

## 1.6. Контактные данные

Всегда создавать отдельную секцию `"contacts"` в `content.json` (даже если секция contacts уже есть).
Найти по всему лендингу: телефон, email, адрес, ссылки на соцсети.
Ключи: `phone`, `email`, `address`, `social_vk`, `social_tg`, `social_wa`, `social_fb`, `social_ig`.
